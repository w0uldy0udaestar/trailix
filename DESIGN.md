# DESIGN.md — range (가칭) 디자인 시스템

> 2026-07-07 /plan-design-review에서 확정. 카드 UI(터미널·hook·채팅)의 단일 기준.
> 원칙: APP UI — 차분한 위계, 적은 색, 유틸리티 언어. 절제가 기본값.

## 색상·글리프 (시맨틱 3+1)

- `✓` 초록 = 통과 · `!` 노랑 = 주의 · `✗` 빨강 = 부실 · `◌`/dim 회색 = 판정 불가·보조
- 시안 = 경로·명령어 (행동 가능 대상)
- **색 = 의미 1:1. 장식색 금지. 이모지 금지** (터미널 폭 불안정 — 텍스트 글리프만)
- NO_COLOR 규약 준수, `--ascii` 폴백 (✓→[OK], !→[!], ✗→[X], ◌→[?])

## 카드 구조 (모든 표면 공통)

박스 헤더(등급 + 한 줄 결론 + "세션 누적 · N턴" 라벨) → 판정(증거 ≤5줄) → 사실 라인(항상 출력) → 판정 불가 룰은 하단 회색 1줄

## 상태

- **정상**: 위 구조 그대로
- **판정 불가**: 차가운 N/A 금지 — "채점할 위임 활동이 아직 부족해요" + 사실 라인 + 다음 안내
- **빈 세션/첫 실행**: 박스 없는 안내문(온보딩 겸) — 작업 후 재실행 유도 + `range list`
- **로딩**: 없음 (즉시 출력이 예산)

## 폭·절단

1. 카드 폭 = min(터미널 폭, 80칸) · 80칸 미만은 박스 생략 평문 모드 · **hook 표면은 min(터미널 폭 − 5, 80)** (⎿ 거터 5칸 — Day 1 스파이크 실측)
2. 한글·전각 = wcwidth 2칸 계산
3. 긴 경로 가운데 생략 `src/…/parser.ts`
4. 파일 ≤3개 + "외 N" · 트리 ≤3레벨 + "+N"
5. 증거 한 줄 ≤76자, 접미 생략

## 언어

메시지 카탈로그 en/ko 분리. 기본 = LANG 자동 감지, `--lang` 강제. 데모 GIF는 영어판.

## 표면별 제약

| 표면 | 형식 |

| CLI (TTY) | ANSI 색 + 유니코드. TTY 감지 |
